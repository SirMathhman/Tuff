# Language Spec — One Page ⚙️🔒

## 1) Purpose & Goals

- **Name (working):** Untitled systems language
- **Primary goal:** Prevent insecure/unsafe code generated by agents for systems/embedded targets (prevent buffer overflows, use-after-free, data races, memory leaks).
- **Priorities:** **Safety** (guiding), **Performance**, **Expressive types**, **Interop (C/LLVM)**.
- **Guiding priority:** **Safety** — design trade-offs favor stronger static guarantees even if that affects ergonomics or code size.

---

## 2) Design Summary

- **Typing:** Static, **dependent/linear** types with ML-like inference.
- **Memory:** **Zero-runtime**, **linear/move** semantics (no GC). Ownership enforced by the type system; no implicit runtime checks required for safety claims unless annotated.
- **Verification:** **SMT-backed**, mostly-inferred proofs; `require`/`ensure`/`assert` annotations for explicit invariants.
- **Concurrency:** Ownership-based across threads + linear types to statically prevent data races. The **async/futures** model is the primary concurrency abstraction to be supported and verified in the stdlib; message-passing channels and actor frameworks can be layered on top.
- **Interop/Targets:** `extern` C-style FFI with **annotated verification contracts** for extern declarations (pre/post conditions checked at compile time). LLVM backend, targets: bare-metal, Linux, Wasm, RTOS. An initial JS compilation path is planned.
- **Ergonomics:** Expression-oriented, **braced** blocks, `:` style type annotations, concise syntax, rich stdlib for systems.

> Note: Compiler prioritizes helpful diagnostics and fast incremental cycles; proofs are automatic where possible, with explicit hints when needed.

---

## 3) Core Syntax (examples)

- Bindings & functions:
  - `let x : I32 = 0;`
  - `fn add(a : I32, b : I32) : I32 => a + b;`
- Structs / methods:
  - `struct Point { x : I32; y : I32; }`
  - `impl Point { fn manhattan(&this) : I32 => abs(this.x) + abs(this.y); }`
- Externs:
  - `extern fn c_get() : I32;`
- Match / enums:
  - `enum Opt[T] { None, Some(T) }`
  - `match opt { Some(v) => v, None => 0 }`
- Verification annotations:
  - `fn safe_read(buf: &Buffer, idx: USize) : I8 requires idx < buf.len ensures true => ...`

---

## 4) Type System & Safety

- **Linear types / move semantics** for resources: values with linear kind must be consumed/moved; compiler enforces single-owner rules.
- **Dependent fragments**: types can depend on values for sizes and bounds, e.g., `Buffer[n : USize]`.
- **SMT integration:** solver drives discharge of assertions and refinement obligations; the compiler **automatically infers proofs** and does not rely on user-authored proof hints in source. If the compiler cannot infer a required proof, compilation fails with clear diagnostics and suggested next steps.
- **Safety properties enforced statically:** bounds, lifetime/moves (no use-after-free), race-freedom (no shared mutable aliasing), memory-leak freedom by default patterns + optional proof obligations for complex lifetimes.

---

## 5) Memory & Concurrency Model

- **No GC**, minimal runtime support for platform-required demarcation only.
- **Ownership model:** linear/move with **auto-borrow and inference by default** to improve ergonomics; explicit move/borrow annotations are available when precise control is required. Inference is predictable and silent by default; diagnostic reporting can be enabled via project settings or compiler flags if desired.
- **Message-passing / channels** in stdlib built on top of ownership rules (safe by construction).

---

## 6) Verification Workflow

- **Full verification by default**: proofs for dependent and linear obligations are required and failures block compilation.
- **Proof model**: proofs are inferred automatically by the compiler; source-level proof hints are intentionally omitted. The compiler will provide diagnostics and suggested lemmas when it cannot discharge obligations; unresolved obligations block compilation by default.
- **Selective verification**: bypassing full verification is intentionally restricted; there is no easy per-module escape hatch — bypasses require explicit compiler flags and are discouraged.
- **Runtime checks**: optional `--sanitize` mode to add checks when proofs are incomplete or when building with a non-default verification mode.

---

## 7) Tooling & Ergonomics

- Fast incremental compiler, **LSP** server, formatter, and REPL.
- Errors: concise primary message + short actionable fix suggestions + `--explain` for extended diagnostics.
- Small but useful standard library with **buffers & slices prioritized for a verified bootstrap** (minimal API first), followed by a **full verified allocator** and device and concurrency primitives (async/futures).

## 8) Minimal EBNF (core)

```
program     ::= { item }
item        ::= fn_decl | struct_decl | enum_decl | extern_decl
fn_decl     ::= "fn" ident "(" [param_list] ")" [":" type] "=>" expr ";"
param_list  ::= param { "," param }
param       ::= ident ":" type
struct_decl ::= "struct" ident "{" { field } "}"
field       ::= ident ":" type ";"
enum_decl   ::= "enum" ident "{" { variant } "}"
extern_decl ::= "extern" "fn" ident "(" [param_list] ")" ":" type ";"
expr        ::= literal | ident | "match" expr "{" case_list "}" | block | call
```

## Decisions (recorded defaults)

- **Guiding priority:** **Safety** — design trade-offs favor stronger static guarantees.
- **Verification default:** **Full** — proofs required by default; bypassing full verification is restricted (no easy per-module escape hatch; bypasses require explicit compiler flags and are discouraged).
- **Proof model:** compiler-inferred proofs; source-level proof hints are intentionally omitted. If the compiler cannot infer a proof, compilation fails with clear diagnostics and suggested next steps.
- **Diagnostics:** compiler inference is silent/predictable by default (no warnings/errors for inferred moves/borrows); diagnostics can be enabled through project settings or compiler flags.
- **Memory ergonomics:** **Auto-borrow/inference** by default; explicit annotations available for precise control.
- **Concurrency primitive:** **Async/futures** model prioritized for verified concurrency; channels & actor frameworks will be built on top.
- **Allocator:** **Full verified allocator** planned as part of the initial stdlib.
- **Stdlib bootstrap priority:** **Buffers & Slices** (verified first) — initial API scope: minimal verified slice view + indexing + length/bounds proofs.
- **Toolchain/targets:** **LLVM/clang compatibility prioritized**; note: an initial JS compilation path is planned.
